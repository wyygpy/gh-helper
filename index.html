使用方法

Git Clone 加速

将仓库地址替换为加速地址，直接 clone：

终端命令
# 原命令 git clone https://github.com/user/repo.git # 加速命令（在仓库地址前加上加速服务地址） git clone https://gh-proxy.example.com/https://github.com/user/repo.git

Wget 下载

使用 wget 下载文件：

终端命令
# 下载 Release 文件 wget https://gh-proxy.example.com/https://github.com/user/repo/releases/download/v1.0/file.zip # 下载仓库归档 wget https://gh-proxy.example.com/https://github.com/user/repo/archive/main.zip # 下载 Raw 文件 wget https://gh-proxy.example.com/https://raw.githubusercontent.com/user/repo/main/file.md

Curl 下载

使用 curl 下载文件：

终端命令
# 下载文件并保存 curl -L -o file.zip https://gh-proxy.example.com/https://github.com/user/repo/releases/download/v1.0/file.zip # 查看 Raw 文件内容 curl -L https://gh-proxy.example.com/https://raw.githubusercontent.com/user/repo/main/README.md # 静默下载 curl -sL -o archive.zip https://gh-proxy.example.com/https://github.com/user/repo/archive/main.zip

浏览器直接下载

直接在浏览器地址栏访问加速链接即可下载：

URL 格式
# 格式：加速服务地址 + GitHub 原始链接 # 下载 Release 文件 https://gh-proxy.example.com/https://github.com/user/repo/releases/download/v1.0/file.zip # 下载仓库归档 https://gh-proxy.example.com/https://github.com/user/repo/archive/refs/heads/main.zip # 查看 Raw 文件 https://gh-proxy.example.com/https://github.com/user/repo/blob/main/README.md

包管理器配置

配置 npm/pip 等使用加速镜像：

配置示例
# Go mod（在 go.mod 中替换） replace github.com/user/repo => https://gh-proxy.example.com/https://github.com/user/repo.git # Git 配置全局替换 git config --global url."https://gh-proxy.example.com/https://github.com/".insteadOf "https://github.com/" # Dockerfile 中使用 RUN curl -L -o app.zip https://gh-proxy.example.com/https://github.com/user/repo/releases/download/v1.0/app.zip

提示与注意事项

  • 仅支持公开仓库，私有仓库无法加速
  • 大文件下载可能较慢，请耐心等待
  • 如遇 404 错误，请检查链接是否正确或文件是否存在
  • Git clone 时需要完整保留 https:// 前缀
